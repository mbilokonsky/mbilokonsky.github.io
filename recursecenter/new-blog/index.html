Ok, new blog! I've been at Recurse Center for the past month and have bounced around between various endeavors.

So far I've:

  • taught a weekly javascript workshop covering topics like Webpack/Babel, React, DevTools, Node.js, etc.
  • learned a bit about machine learning (enough to know that it's not something I want to break into just yet),
  • spent some time trying to extend the javascript language (I want to add an isolated keyword which would allow you to assert that a function makes zero use of external state) by writing a babel plugin, but I found that you can't use babel for something like this because the underlying parser that babel uses (babylon) can't be extended without official support from the ECMAScript standards body
  • defined and spec'd out a node.js tool for creating and hosting chat bots across a variety of services, only to learn that hubot has basically already implemented everything I was going to do. Whoops.
  • did some python, to extract colors from images (for use in a larger toolchain around video processing)
  • set up this blog, and
  • decided on how I want to spend the bulk of the rest of my time at RC: by writing a language.

So, the plan is to write a language that will compile into javascript, but with a very limited scope. This language (tentatively called "reduxion") is for defining a redux store (with associated reducers, action creators, etc) in a concise and elegant way. It'll eliminate a lot of boilerplate redux code, but more importantly it'll teach me how to work with parsers and lexers and compilers and the whole thing from bottom to top.

I've chosen to set up this blog using metalsmith in part because metalsmith is highly extensible, and it should be simple enough for me to create some interesting interactive posts to play with fun techniques. :)
